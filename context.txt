## Project Context for Next Chat Session

### What this project is
CS2 (Counter-Strike 2) Inventory Manager — a Flutter mobile app that fetches a user's Steam inventory, shows items with images/rarity/prices from multiple sources, and will eventually alert on price spikes. See cs2_app.md for the full 6-phase plan.

- **Phase 3: COMPLETE** — Steam API integration working. Inventory fetches with progress UI, caches to disk, persists across restarts.

### What was built/changed this session

#### HttpOnly cookie fix — SOLVED
- **Problem**: `steamLoginSecure` cookie is HttpOnly — JavaScript `document.cookie` can't read it
- **Solution**: Native Android platform channel in `MainActivity.kt` calls `android.webkit.CookieManager.getInstance().getCookie(url)` which CAN read HttpOnly cookies
- `steam_login_screen.dart` calls `MethodChannel('com.deportfolio/cookies').invokeMethod('getCookies', {'url': url})` to get the cookie
- Cookie is now automatically extracted on login — no manual paste needed
- Settings screen shows green "Session cookie saved" after login

#### Price history: INR→USD conversion
- **Problem**: Steam's pricehistory endpoint ignores the `currency` parameter and returns prices in the account's local currency (INR for Indian accounts)
- **Solution**: `price_history_service.dart` fetches exchange rate from `open.er-api.com/v6/latest/USD` (free, no API key), caches rate to disk for 24h, divides all prices by INR/USD rate
- Fallback rate of 85.0 if API unreachable
- Cache version system (currently v5) — bump to invalidate old caches with bad data

#### Price history: hourly vs daily granularity
- Service now returns **hourly** points (no server-side aggregation)
- Chart widget decides granularity: 7D/1M with ≥10 avg trades/day → hourly; otherwise → daily (median)
- Daily aggregation uses median price (not average) for resistance to outlier trades

#### Price chart: fl_chart native zoom/pan
- Uses `FlTransformationConfig(scaleAxis: FlScaleAxis.horizontal, minScale: 1.0, maxScale: 25.0)`
- Native pinch-to-zoom and single-finger pan — no custom gesture code needed
- Range chips (7D/1M/3M/6M/1Y/All) filter the data and reset zoom to 1x
- User can then pinch-to-zoom within the selected range
- Sticky tooltips: tap to place, stays visible after lifting finger
- Bottom axis date format adapts: `d/M HH:mm` for hourly ≤7d, `MMM d` for ≤90d, `MMM yy` for longer
- Left axis: compact format for large prices (`$1.2k` instead of `$1200.00`)

#### Auto-fetch disabled
- `home_screen.dart` no longer auto-fetches prices on startup/hot restart
- Prevents accidental rate limit issues during development
- User must manually tap fetch buttons

#### Pull-to-refresh removed from inventory screen
- Was accidentally triggering full inventory re-fetch from Steam
- Fetch inventory now only available from Settings → "Fetch Inventory" button

#### Clear price history cache button
- Added to Settings screen between cookie card and currency card
- Calls `priceHistoryService.clearCache()` which deletes the `price_history/` directory

### Important notes
- **Impeller disabled** in AndroidManifest.xml — required for WebView on Samsung S24 Ultra
- **Riverpod v3**: no StateProvider, no valueOrNull, watch outside .when(), use Notifier with set()
- **Rate limits**: 1 sec between requests for both Steam Market and CSFloat
- **Price history currency**: Steam returns INR regardless of currency param — we convert to USD via exchange rate API
- **Cache versioning**: bump `_cacheVersion` in price_history_service.dart to invalidate old cached data
- **dev.log doesn't show in console** — use `debugPrint` instead for visible output
- **APK build**: `flutter build apk --release` produces APK at `build/app/outputs/flutter-apk/app-release.apk`
