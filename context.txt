## Project Context for Next Chat Session

### What this project is
CS2 (Counter-Strike 2) Inventory Manager — a Flutter mobile app that fetches a user's Steam inventory, shows items with images/rarity/prices from multiple sources, and will eventually alert on price spikes. See cs2_app.md for the full 6-phase plan.

### Current state
- **Phase 1: COMPLETE** — Full UI shell with dark theme, bottom nav (GoRouter), 4 screens, Riverpod state management, filter system, sort options.
- **Phase 3: COMPLETE** — Steam API integration working. Inventory fetches with progress UI, caches to disk, persists across restarts.
- **Phase 4: COMPLETE** — Steam Market + CSFloat pricing, price history charts with fl_chart, rate limits.
- **Phase 2: COMPLETE** — Steam WebView login, Firebase Auth via Cloud Function custom token, Firestore inventory sync with change detection and quota protection.
- **Phase 6: COMPLETE** — Node.js storage service deployed to GCE, Flutter UI working standalone on device, pricing + images + collection data working for storage items. App is fully standalone (no laptop needed).

#### Inventory float lookup via GC
- Added `GET /inventory/floats` endpoint to Node.js storage service — returns float values for all inventory items directly from GC data (no inspect requests needed for own items)
- Added `GET /inspect?url=...` endpoint — resolves float from any inspect link via GC (for future use)
- Added `getInventoryFloats()` to Flutter `StorageService`
- Added `fetchInventoryFloats()` to `InventoryNotifier` — merges float data into inventory items and caches
- Added float fetch button (water drop icon) to inventory screen app bar
- Floats auto-fetch from GC after every Steam inventory refresh
- Floats are cached to disk so they persist across restarts

#### Steam login WebView fix
- Fixed loop where going to Steam app for Guard code would reset the WebView to the login page
- Root cause: `didChangeAppLifecycleState` called `_extractAndClose()` on any resume, even before auth completed
- Fix: `_onResumedFromBackground()` checks current URL first — only extracts if already past login page
- `_navigatedToProfile` flag prevents double-navigation to `steamcommunity.com/my/profile`
- `_tryExtract()` has `_extracting` guard to prevent concurrent extraction attempts
- "Done" button calls `_goToProfileAndExtract()` which navigates to steamcommunity if not already there

#### GC auto-reconnect
- Added reconnect logic in `disconnectedFromGC` handler in `storage-service/index.js`
- If Steam session is still alive when GC drops, re-sends `gamesPlayed([730])` after 10s to prompt reconnect

#### Storage unit CSFloat total
- Added `totalCsfloatValue` field to `StorageUnit` model
- Calculated wherever `totalValue` is calculated (cache load, fetchContents, _mergeItemUpdates with recalcValue=true)
- Storage unit subtitle now shows two lines: item count, then `Steam $X  ·  CSFloat $Y`

#### Background price refresh (WorkManager)
- Added `workmanager: ^0.9.0` dependency
- `lib/background/price_refresh_task.dart` — standalone background task (no Riverpod): reads inventory + storage caches from disk, fetches Steam + CSFloat prices via service classes, writes back
- Registered as periodic task in `main.dart` with 6h frequency, aligned to clock slots (12 AM, 6 AM, 12 PM, 6 PM) via `initialDelay` calculation
- `callbackDispatcher` is top-level with `@pragma('vm:entry-point')` annotation
- `saveLastPriceFetchTimestamp()` (top-level fn in price_provider.dart) called after inventory fetch, storage fetch, and background refresh
- Home screen shows "Prices updated today at 6:00 PM" (exact timestamp, not relative) below price fetch cards

#### Daily auto-refresh on app open
- `PriceFetchNotifier.autoRefreshIfStale()` checks `last_price_fetch.txt` timestamp — triggers inventory price fetch if >24h old
- Called from `HomeScreen.initState()` via `addPostFrameCallback`
- Storage units auto-refresh prices when expanded if their cache is >24h old (`_isCacheStale()` in storage_provider)

#### Price chart minimap with viewport navigation and manual zoom/pan
- Removed fl_chart's `FlTransformationConfig` — replaced with custom `GestureDetector` zoom/pan using `_viewStart`/`_viewEnd` (0–1 fractions of full daily timeline)
- `_Minimap` widget below chart always shows entire dataset; blue viewport rectangle tracks visible window
- Pinch-to-zoom: uses `details.pointerCount >= 2` to detect, `_baseViewWidth / details.scale` for zoom factor, `IgnorePointer` wraps `LineChart` to prevent fl_chart from stealing gestures
- Single-finger drag pans the chart; `Listener` widget wraps `GestureDetector` for reliable tap-to-tooltip detection
- Tap or drag on minimap to navigate — viewport slides and main chart follows
- Range chips auto-switch based on visible day count (`_updateSelectedRange()`) when zooming
- Dynamic minimum zoom window (`_minWindow`) calculated from data length — allows zooming down to ~6 hours of data
- Range chip selection calls `_applyRange()` which sets the window to show the most recent N days

### Important notes
- **Impeller disabled** in AndroidManifest.xml — required for WebView on Samsung S24 Ultra (deprecation warning shown)
- **Riverpod v3**: no StateProvider, no valueOrNull, watch outside .when(), use Notifier with set()
- **Rate limits**: Steam search/render 3s between requests; CSFloat 1s; both retry on 429 with backoff
- **Steam search/render quirk**: `|` in query is treated as OR — wrap query in quotes for exact match
- **Price history currency**: Steam returns INR regardless of currency param — we convert to USD via exchange rate API
- **Cache versioning**: bump `_cacheVersion` in price_history_service.dart to invalidate old cached data
- **dev.log doesn't show in console** — use `debugPrint` instead for visible output
- **APK build**: `flutter build apk --release` produces APK at `build/app/outputs/flutter-apk/app-release.apk`
- **Firestore free tier**: 20k writes/day. Change detection keeps normal usage at ~200-300 writes/session
- **Steam Guard login**: use SteamClient platform type, pass code via fresh session to avoid poll timeout
- **CSFloat API key async load**: fetch waits up to 3s for key to load from disk before starting
- **Parallel price fetch race condition**: solved with `_mergeItemUpdates()` that reads current state before applying, plus separate tracking keys per fetch type
- **Firebase Auth persists**: sessions survive app restart. Only need to re-login if Firebase session expires (rare) or user signs out.
- **Steam login cookie expires**: separate from Firebase Auth. Cookie needed for price history charts. Settings page validates cookie on load and shows expired status.
- **Cloud Function deployed**: `createCustomToken` on us-central1, project cs2-portfolio (Blaze plan), `invoker: "public"` required for Gen 2 callable functions
- **Cloud Function IAM**: compute service account needs `Service Usage Consumer` + `Service Account Token Creator` roles for `createCustomToken()` to work
- **PowerShell note**: use `;` not `&&` to chain commands
- **LoggedInElsewhere**: Steam only allows one session "playing" a game at a time. The VM service uses `playingState` handler to yield when real Steam client is active. Also auto-reconnects after 30s if kicked. Don't run local `node index.js` while VM service is running.
- **GCE ephemeral IP**: 34.44.97.110 may change on VM restart. Reserve a static IP in GCE to make it permanent (free for attached VMs).
- **GCE browser SSH**: long commands get mangled by line wrapping. Use short commands or push files via git instead.
