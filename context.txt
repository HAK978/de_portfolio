### Key issue: HttpOnly cookie
The `steamLoginSecure` cookie is HttpOnly — JavaScript in the WebView cannot read it. The WebView login successfully:
- Gets the Steam ID automatically
- Establishes a valid session (verified by XHR test to price history endpoint)
But it CANNOT extract the raw cookie value for use in the Dart HTTP client.

**Current workaround:** User manually copies `steamLoginSecure` from browser DevTools and pastes it in Settings → "Steam Login Cookie" field.

**Possible solutions for next session:**
1. Write a native Android platform channel to call `android.webkit.CookieManager.getCookie(url)` which CAN read HttpOnly cookies
2. Route price history requests through the WebView instead of Dart's HTTP client
3. Use `flutter_inappwebview` package which has full cookie access API
4. Accept manual paste as the permanent solution
